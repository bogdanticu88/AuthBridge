# Contributing to AuthBridge

First off, thank you for considering contributing to AuthBridge! It's people like you that make AuthBridge such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub issue tracker to report bugs.
- Describe the bug in detail and provide a reproduction script if possible.

### Suggesting Enhancements
- If you have an idea for a new feature, please open an issue to discuss it before submitting a PR.

### Submitting a Pull Request
1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`go test ./...`).
5. Make sure your code lints (`golangci-lint run`).

## Style Guide
- Follow standard Go idioms.
- Use `zerolog` for logging.
- Ensure all exported functions have comments.
